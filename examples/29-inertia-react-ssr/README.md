# 29 · Inertia + React (SSR)

Inertia.js v3 with **React** in **server-side rendered** mode. This
is the "Laravel + Inertia (SSR)" recipe: the server renders the React
page tree to HTML, the client **hydrates** the existing DOM.

## What it shows

- `createReactAdapter` SSR adapter plugged into `new Inertia({ ssr })`
- First-page HTML is fully server-rendered; `<div id="app">` contains
  the actual React tree, not just a `data-page` JSON placeholder.
- Client uses `hydrateRoot` (NOT `createRoot`) so the existing
  server-rendered DOM is reused.
- Same page component (`HomePage`) is shared between server and client.

## How to run

```bash
cd examples/29-inertia-react-ssr
bun install                           # install react/react-dom peer-deps (workspace root)
bun build ./frontend/client.tsx \
  --outdir=./public --target=browser --format=esm --minify
bun main.ts
```

Open <http://localhost:3000> and view-source. The response body
contains the rendered `<h1>Hello from Inertia + React SSR!</h1>`,
not a `data-page` JSON string.

## Dev mode

```bash
# Terminal 1: rebuild the hydration client on every change
bun build ./frontend/client.tsx \
  --outdir=./public --target=browser --format=esm --watch

# Terminal 2: run the server
bun main.ts
```

## Code

### Server — `main.ts`

```ts
import "reflect-metadata";
import path from "node:path";
import { Application, Module, Controller, Get, Post, Body, Inject, Injectable } from "@nexusts/core";
import { StaticModule } from "@nexusts/static";
import { Inertia, createReactAdapter } from "@nexusts/view";
import { HomePage } from "./frontend/home.tsx";

@Injectable()
@Controller("/")
class HomeController {
  @Inject(Inertia.TOKEN) declare private inertia: Inertia;

  @Get("/")
  home() { return this.inertia.render("Home", { greeting: "...", count: 0 }); }

  @Post("/counter")
  counter() { return this.inertia.location("/"); }

  @Post("/greet")
  greet(@Body() body: { name?: string }) {
    if (!body?.name) {
      c.status(422);
      return this.inertia.render("Home", { ..., errors: { name: "Required" } });
    }
    return this.inertia.render("Home", { greeting: `Welcome, ${body.name}!` });
  }
}

@Module({
  providers: [{
    provide: Inertia.TOKEN,
    useValue: new Inertia({
      ssr: createReactAdapter({ components: { Home: HomePage } }),
    }),
  }],
  controllers: [HomeController],
})
class AppModule {}

const app = new Application(AppModule);
app.server.app.use("/static/*", StaticModule.mount({
  root: path.join(import.meta.dir, "public"),
  prefix: "/static",
}));

const port = Number(process.env.PORT ?? 3000);
await app.listen(port);
```

### Shared page component — `frontend/home.tsx`

```tsx
export function HomePage({ greeting, count, errors }: any) {
  return (
    <main>
      <h1>{greeting}</h1>
      <p>Counter: {count}</p>
      <a href="/counter" method="post" as="button">+1</a>
      <form action="/greet" method="post">
        <input name="name" />
        {errors?.name && <p style={{ color: "crimson" }}>{errors.name}</p>}
        <button>Submit</button>
      </form>
    </main>
  );
}
```

### Hydration entry — `frontend/client.tsx`

```tsx
import { createInertiaApp } from "@inertiajs/react";
import { hydrateRoot } from "react-dom/client";   // ← hydrateRoot, not createRoot
import { HomePage } from "./home.tsx";

createInertiaApp({
  resolve: (name) => HomePage,
  setup({ el, App, props }) {
    hydrateRoot(el, <App {...props} />);
  },
});
```

## How to verify SSR is working

```bash
curl -s http://localhost:3000/ | grep -A 1 "<div id=\"app\""
```

If the response contains your component's HTML **inside** `<div id="app">`,
SSR is working. If you only see `data-page="..."` with an empty `<div>`,
the SSR adapter failed and the framework fell back to the default shell
(client-only hydration).

## CSR variant

For client-side only rendering (no SSR), see
**`examples/28-inertia-react-spa`**. The CSR variant doesn't require
`react`/`react-dom` on the server and has a slightly simpler build
pipeline (no `home.tsx` import in `main.ts`).

## Smoke test note

The boot test verifies the server starts and serves a 200 response.
It does **not** verify that SSR actually rendered (would require
parsing the HTML). Run `curl http://localhost:3000/` once after
`bun build` to confirm the SSR HTML is present.
