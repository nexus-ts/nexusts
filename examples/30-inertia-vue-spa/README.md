# 30 · Inertia + Vue 3 (SPA)

Inertia.js v3 with **Vue 3** in client-side-only mode. The
"Laravel + Inertia (CSR) with Vue" recipe.

## What it shows

- `createInertiaApp` from `@inertiajs/vue3`
- `createApp(...).use(plugin).mount(el)` for the SPA root
- Vue SFC-style template strings (no JSX, no `.vue` SFC compiler needed)
- Same controller contract as the React variants (28, 29)

## How to run

```bash
cd examples/30-inertia-vue-spa
bun install                                  # install vue + @inertiajs/vue3 (workspace root)
bun build ./frontend/app.js --outdir=./public \
  --target=browser --format=esm --minify
bun main.ts
```

Open <http://localhost:3000>.

## Dev mode

```bash
# Terminal 1: rebuild the client on save
bun build ./frontend/app.js --outdir=./public \
  --target=browser --format=esm --watch

# Terminal 2: run the server
bun main.ts
```

## Code

### Server — `main.ts`

```ts
import "reflect-metadata";
import path from "node:path";
import { Application, Module, Controller, Get, Post, Body, Inject, Injectable } from "@nexusts/core";
import { StaticModule } from "@nexusts/static";
import { Inertia } from "@nexusts/view";

@Injectable()
@Controller("/")
class HomeController {
  @Inject(Inertia.TOKEN) declare private inertia: Inertia;

  @Get("/")
  home() { return this.inertia.render("Home", { greeting: "...", count: 0 }); }

  @Post("/counter")
  counter() { return this.inertia.location("/"); }

  @Post("/greet")
  greet(@Body() body: { name?: string }) {
    if (!body?.name) { c.status(422); return this.inertia.render("Home", { ..., errors: { name: "Required" } }); }
    return this.inertia.render("Home", { greeting: `Welcome, ${body.name}!` });
  }
}

@Module({
  providers: [{ provide: Inertia.TOKEN, useValue: new Inertia() }],
  controllers: [HomeController],
})
class AppModule {}

const app = new Application(AppModule);
app.server.app.use("/static/*", StaticModule.mount({
  root: path.join(import.meta.dir, "public"),
  prefix: "/static",
}));

const port = Number(process.env.PORT ?? 3000);
await app.listen(port);
```

### Client — `frontend/app.js`

```js
import { createApp, h } from "vue";
import { createInertiaApp } from "@inertiajs/vue3";

createInertiaApp({
  resolve: (name) => Home,                       // map "Home" → local component
  setup({ el, App, props, plugin }) {
    createApp({ render: () => h(App, props) })
      .use(plugin)
      .mount(el);
  },
});

const Home = {
  props: ["greeting", "count", "errors"],
  template: `
    <main>
      <h1>{{ greeting }}</h1>
      <p>Counter: {{ count }}</p>
      <a href="/counter" method="post" as="button">+1</a>
      <form action="/greet" method="post">
        <input name="name" />
        <p v-if="errors?.name" style="color: crimson;">{{ errors.name }}</p>
        <button>Submit</button>
      </form>
    </main>
  `,
};
```

## React vs Vue

| Concern | React (28, 29) | Vue 3 (30, 31) |
| ------- | -------------- | -------------- |
| Client lib | `react`, `react-dom` | `vue` |
| Adapter pkg | `@inertiajs/react` | `@inertiajs/vue3` |
| Component format | `.tsx` (JSX) | `.js` (template string) |
| SSR engine | `react-dom/server` | `@vue/server-renderer` |
| Mount | `createRoot` / `hydrateRoot` | `createApp().use(plugin).mount()` |

The **server-side code is identical** across all four examples —
only the client and the optional SSR adapter change.
