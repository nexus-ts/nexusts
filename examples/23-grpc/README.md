# 23 · gRPC

gRPC server + typed client with reflection-based service discovery
(`@nexusts/grpc`).

## What it shows

- Define services with `@GrpcService()`
- `@GrpcMethod('MethodName')` for handlers
- gRPC client via `GrpcServiceClient`
- `.proto` files loaded at runtime (no codegen)

## How to run

```bash
cd examples/23-grpc
bun main.ts
```

Server listens on `localhost:50051` for the `greeter.Greeter` service.

## Proto

`proto/greeter.proto`:

```proto
syntax = "proto3";
package greeter;

service Greeter {
  rpc SayHello (HelloRequest) returns (HelloReply) {}
}

message HelloRequest { string name = 1; }
message HelloReply { string message = 1; }
```

## Code

```ts
import "reflect-metadata";
import path from "node:path";
import { Module, Injectable, Inject } from "@nexusts/core";
import {
  GrpcModule, GrpcService, GRPC_SERVICE_TOKEN,
  GrpcServiceDecorator, GrpcMethod,
} from "@nexusts/grpc";

// Server side
@Injectable()
@GrpcServiceDecorator("GreeterService")
class GreeterServiceImpl {
  @GrpcMethod("SayHello")
  sayHello(request: { name: string }) {
    return { message: `Hello, ${request.name}!` };
  }
}

@Module({
  imports: [
    GrpcModule.forRoot({
      protoPath: "./proto/greeter.proto",
      host: "0.0.0.0",
      port: 50051,
      services: [GreeterServiceImpl],
    }),
  ],
  providers: [GreeterServiceImpl],
})
class GrpcAppModule {}
```

## gRPC client (in another service)

```ts
@Injectable()
class GreeterClient {
  @Inject(GRPC_SERVICE_TOKEN) declare grpc: GrpcService;
  private client: any;
  constructor() {
    this.client = this.grpc.client("GreeterService", { url: "localhost:50051" });
  }
  async sayHello(name: string) {
    return this.client.SayHello({ name });
  }
}
```

## Reflection-based

The client is generated by parsing the `.proto` at server start.
Once the gRPC server is running, the `client(name, { url })` method
returns an object whose methods match the gRPC service methods.
No codegen step required.

## Unary only (v1)

Stream methods (`stream` keyword) are not yet supported — they will be
added in a later release.
