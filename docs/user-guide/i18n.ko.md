# 국제화 · `@nexusts/i18n` (v0.5)

> English: [`i18n.md`](./i18n.md)
> v0.5 신규. Bun 네이티브 스택을 위한 locale 인식 번역 및 포맷팅.
> 외부 의존성 0 — Node의 내장 `Intl` API 사용.

`@nexusts/i18n`이 제공하는 것:

- **`I18nService`** — 메시지 번역, `Intl`로 날짜/숫자/통화 포맷.
- **`I18nModule.forRoot(config)`** — 서비스를 DI 컨테이너에 연결. 선택적으로 디렉토리에서 JSON 파일 로드.
- **`i18nMiddleware(service)`** — Hono 미들웨어. 요청별로 `?lang=` → `lang` 쿠키 → `Accept-Language` → default 순으로 활성 locale 감지.
- **`@CurrentLocale()`** — 활성 locale 문자열을 주입하는 컨트롤러 파라미터 데코레이터.

---

## 1. 빠른 시작

```ts
import { Module, Inject } from "@nexusts/core";
import { I18nModule, I18nService, I18N_SERVICE_TOKEN, CurrentLocale } from "@nexusts/i18n";

@Module({
  imports: [
    I18nModule.forRoot({
      defaultLocale: "en",
      messages: {
        en: {
          hello: "Hello, :name!",
          "auth.errors.invalid": "Invalid email or password",
        },
        ko: {
          hello: "안녕하세요, :name님!",
          "auth.errors.invalid": "이메일 또는 비밀번호가 올바르지 않습니다",
        },
      },
    }),
  ],
})
class AppModule {}

@Injectable()
class UserService {
  @Inject(I18N_SERVICE_TOKEN) declare i18n: I18nService;

  greet(name: string, locale: string) {
    return this.i18n.t("hello", { name }, locale);
  }
}

@Controller("/users")
class UserController {
  @Get("/")
  index(@CurrentLocale() locale: string) {
    return { greeting: "Welcome", locale };
  }
}
```

## 2. `t()` 함수

```ts
i18n.t("hello", { name: "Alice" });            // → "Hello, Alice!" (default locale)
i18n.t("hello", { name: "Alice" }, "ko");     // → "안녕하세요, Alice님!"
i18n.t("missing.key");                         // → "missing.key" (원시 키, dev가 볼 수 있음)
i18n.tOr("missing", "fallback text");          // → "fallback text"
```

### 보간

`:name` 스타일 placeholder 사용. 알 수 없는 placeholder는 그대로 남음 (그래서 dev가 렌더링된 출력에서 `:missing`을 볼 수 있음).

```ts
"profile.welcome": "Welcome, :name! You have :count notifications."
i18n.t("profile.welcome", { name: "Alice", count: 5 });
// → "Welcome, Alice! You have 5 notifications."
```

### `|`로 복수형 처리

`|` 문자는 메시지를 복수형 세그먼트로 분리. 세그먼트 수는 CLDR 복수 카테고리에 매핑:

| 세그먼트 | 카테고리 (순서대로) |
| -------- | --------------------- |
| 1 | other |
| 2 | one \| other |
| 3 | zero \| one \| other |
| 4 | zero \| one \| two \| other |
| 5 | zero \| one \| two \| few \| other |
| 6 | zero \| one \| two \| few \| many \| other |

```ts
// 영어: 0 → "other", 1 → "one", 2+ → "other"
// 따라서 2-세그먼트 폼이 모든 케이스 처리:
"items.count": "1 item|:count items"

i18n.t("items.count", { count: 0 });  // → "0 items"
i18n.t("items.count", { count: 1 });  // → "1 item"
i18n.t("items.count", { count: 5 });  // → "5 items"
```

프레임워크는 `Intl.PluralRules`를 사용하므로 언어별 복수 처리가 자동 — 영어 1은 `one`, 나머지는 `other`. 슬라브어/아랍어/웨일스어 등은 locale별 규칙 적용.

### 중첩 키

dot 표기로 메시지 dict 순회:

```ts
messages: {
  en: {
    auth: {
      welcome: "Welcome, :name!",
      errors: { invalid: "Invalid credentials" },
    },
  },
}

i18n.t("auth.welcome", { name: "Alice" });        // → "Welcome, Alice!"
i18n.t("auth.errors.invalid");                    // → "Invalid credentials"
```

## 3. Locale 폴백

요청한 locale에 키가 없으면 서비스는 다음 순서로 폴백:

1. 지역을 제거한 같은 locale (`fr-CA` → `fr`)
2. Default locale (설정 가능)
3. 키 자체 (dev 가시)

```ts
const i18n = new I18nService({
  defaultLocale: "en",
  messages: {
    en: { hi: "hi" },
    fr: { hi: "salut" },
  },
});

i18n.t("hi", undefined, "fr-CA");  // → "salut" (지역 폴백)
i18n.t("hi", undefined, "ko");    // → "hi"   (default 폴백)
i18n.t("missing");                  // → "missing"
```

엄격한 locale 강제를 원하면 default-locale 폴백 비활성화:

```ts
const i18n = new I18nService({ fallbackToDefault: false });
i18n.t("hi", undefined, "ko");     // → "hi" (키 자체, "ko"에 "hi" 없음)
```

## 4. Locale 감지 (미들웨어)

미들웨어는 Hono 앱에 설치. 요청별로 locale를 추출하고 `c.var.locale`에 저장.

```ts
import { Hono } from "hono";
import { i18nMiddleware, I18nService } from "@nexusts/i18n";

const svc = new I18nService({ /* ... */ });
const app = new Hono();
app.use("*", i18nMiddleware(svc));
```

우선순위 (첫 매치 우선):

1. **Query string** — `?lang=ko` (키 설정 가능)
2. **Cookie** — `lang=ko` (키 설정 가능)
3. **`Accept-Language` 헤더** — quality score로 파싱
4. **Default locale**

미들웨어는 편의상 `I18nService` 인스턴스도 `c.var.i18n`에 첨부.

## 5. 포매터

모든 포매터는 Node의 내장 `Intl` API를 사용하고 `Intl.*` 인스턴스를 캐시.

### 날짜

```ts
i18n.formatDate(new Date());                 // default locale
i18n.formatDate(new Date(), { locale: "ko-KR" });
i18n.formatDate(new Date(), {
  locale: "en-US",
  year: "numeric", month: "long", day: "numeric",
});
// → "June 22, 2026"
```

### 숫자

```ts
i18n.formatNumber(1234.56);                  // "1,234.56" (en-US)
i18n.formatNumber(1234.56, { locale: "de-DE" });  // "1.234,56"
```

### 통화

```ts
i18n.formatCurrency(1234.56, { currency: "USD" });            // "$1,234.56"
i18n.formatCurrency(1234567, { currency: "KRW", locale: "ko-KR" });  // "₩1,234,567"
```

### Locale 인식 비교

```ts
const a = ["B", "a", "C"];
const sorted = [...a].sort((x, y) => i18n.compare(x, y));
// ['a', 'B', 'C'] — 대소문자 무시 정렬
```

## 6. 디스크에서 메시지 로드

두 가지 옵션:

**옵션 A**: 인라인 메시지 전달

```ts
I18nModule.forRoot({ messages: { en: {...}, ko: {...} } });
```

**옵션 B**: `*.json` 파일이 있는 디렉토리에서 로드 (Node만)

```
locales/
  en.json    { "hello": "Hello, :name!" }
  ko.json    { "hello": "안녕하세요, :name님!" }
  fr.json    { "hello": "Bonjour, :name !" }
```

```ts
I18nModule.forRoot({
  defaultLocale: "en",
  messagesDir: "./locales",
});
```

각 파일의 이름 (`.json` 제외)이 locale 코드가 됨.

Cloudflare Workers / 다른 edge 런타임에서는 `fs` 모듈 사용 불가 — **옵션 A**를 사용하고 빌드 타임에 메시지를 번들.

## 7. 런타임에 메시지 추가

```ts
const svc = container.resolve(I18nService);

// 새 locale 추가
svc.addMessages("fr", { hello: "Bonjour, :name !" });

// 기존에 추가 (병합) — 플러그인에 유용
svc.addMessages("en", { plugin: { greeting: "Hi from plugin" } });
```

`addMessages`는 중첩 dict를 재귀적으로 병합하므로 locale를 점진적으로 확장 가능.

## 8. 설정

```ts
interface I18nConfig {
  defaultLocale?: Locale;                  // default: "en"
  fallback?: boolean;                     // default: true (지역 → default)
  fallbackToDefault?: boolean;            // default: true
  supportedLocales?: Locale[];            // 비어있으면 모두 허용
  messages?: MessageCatalog;              // 인라인
  messagesDir?: string;                   // *.json 파일 경로
  detectQueryKey?: string;                // default: "lang"
  detectCookieKey?: string;               // default: "lang"
}
```
